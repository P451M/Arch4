# Support

Use GitHub Issues for bugs, installation problems, and feature requests after
the repository is public.

Before opening an issue, run:

```sh
arch4 doctor
```

Include the Arch4 version, platform, Cursor version, command output, and any
relevant diagnostics from `.arch4/architecture/build/diagnostics.json`.

Security reports should follow [SECURITY.md](SECURITY.md), not public issues.
