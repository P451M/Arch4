# Create Arch4 Support Request

Draft a sanitized GitHub issue for Arch4.

Use Arch4 MCP tools only when useful:

1. If this workspace has Arch4 files, call `arch4_diagnostics` and summarize
   relevant diagnostics.
2. Ask for missing reproduction steps or feature workflow details if the request
   is ambiguous.
3. Draft a concise issue for `P451M/Arch4` with type, summary, steps or
   workflow, expected behavior, actual behavior, environment, diagnostics, and
   security check.

Do not include secrets, tokens, credentials, private paths, proprietary source,
or unrelated repository details. Do not edit architecture source.
